About Ypsilon Script

Project Overview

Ypsilon Script (YS) is an open-source programming language designed specifically for microcontroller development. It provides a modern, type-safe syntax that compiles directly to C++ code compatible with Arduino and other microcontroller platforms.

The goal of Ypsilon Script is to make embedded systems programming more accessible while maintaining the performance and efficiency that microcontrollers require. By combining modern language features with hardware-specific abstractions, YS makes it easier to write correct, maintainable code for embedded systems.

Project Goals

  • Accessibility: Make microcontroller programming accessible to more developers
  • Type Safety: Catch errors at compile time with strong static typing
  • Modern Syntax: Provide a clean, readable syntax inspired by modern languages
  • Zero Overhead: Compile to efficient C++ with no runtime performance penalty
  • Hardware Abstraction: Built-in types for common sensors and peripherals
  • Compatibility: Work with existing Arduino libraries and hardware

The Author

Ypsilon Script was created by Youssef Charfi, a developer passionate about making embedded systems programming more approachable and enjoyable. The project started as an exploration of how modern language design principles could be applied to the microcontroller domain.

Technology Stack

Ypsilon Script is built with the following technologies:

  • Language: JavaScript/Node.js for the compiler
  • Target: C++ code compatible with Arduino and AVR-GCC
  • Testing: Jest for unit testing
  • Version Control: Git and GitHub

Design Philosophy

Ypsilon Script follows these core design principles:

  • Explicit over Implicit: All types and behaviors should be clearly stated
  • Safety First: Prevent common programming errors at compile time
  • Hardware-Aware: Understand the constraints and capabilities of microcontrollers
  • Developer-Friendly: Provide clear error messages and helpful tooling
  • Performance Matters: Generate efficient C++ code without runtime overhead
